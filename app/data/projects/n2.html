University Project

N2: University Notices V2

N2 is a user-customizable web-based announcement board for university students.

It was developed with Django, a Python framework, as a university project in a team of three.

Students at the University of Melbourne are provided with many opportunities to be actively involved in clubs and societies, attend events, explore career prospects and get engaged in other activities that enhance their student experience.

It is essential that students have access to a medium of communication for these matters, so that they are always informed of great opportunities.

Unfortunately, the university does not provide a single channel to broadcast all these notices, and instead uses an unorganized mix of legacy systems including the university portal, e-mail, and other disjoint web services.

The Current Siloed Systems

Notices on opportunities and events are provided through many mediums. One of the main channels include the notification tab on the university online portal. This mainly consist of administrator-moderated broadcast messages from clubs of interest around the university, yet they are mixed with official university announcements, which really are of a different nature. What students receive in the portal are filtered based on their residency status, campus location and course of study, and this filter is not user-configurable.

Some other announcements, which come from individual departments in the university, doesn't come through the portal, but instead through the Blackboard learning management system.

Then again, some other notices are posted on the department's on web pages.

Furthermore employment-related events are sometimes placed on the university's career services online site instead.

Occasionally they choose to broadcast notices via e-mail.

Students who simply want to broadcast messages sometimes simply go through a more familiar medium, such as Facebook.

The Problem of Having All These Mediums

There is no easy way for students to find out about all events and opportunities of interest, nor is there a simple way to keep track of all posts that they are interested in. If they remember something that they were interested in and wanted to recall the details, they must also remember on which medium they saw it, otherwise it will be very hard to search for them.

Most channels do not allow embedding of images or video, limiting content to be text-based only. This greatly reduces the appeal of notices.

In addition, not all mediums provide the facility to book into events. For those that do, if they chose to attend an event, they may or may not be reminded of it closer to the date.

Furthermore, there is no easy way to communicate between students and notice broadcasters.

The Development Process

We decided to create a consolidated platform for all notices. Announcements, events, career opportunities, all at one place. It will be created as a web application, accessible in most modern browsers.

It will allow students to filter notices, keep track of those of interest, and post their own notices.

It comes with full moderator support, which allows university staff to approve student-created posts before it is broadcasted.

We are naming it N2, which stands for Notices Version 2. It is developed by myself and two friends, David Andika and Jonathan Wongsodihardjo.

Functionality

Notices are organized using tags. Each tag belongs to a category, and these are used to label notices to simplify searching and filtering.

There are three user levels, each of them has access to the following features:

  1. All Users
    • Browse notices, search by keyword and tags, view bookmarks
    • View notices in detail
    • Like and bookmark notices
    • Create notices, tag them then submit them, subject to moderation
  2. Moderators
    • Manage notices - approve or disapprove submitted notices; edit or delete existing ones
    • Manage tags - add, edit and delete tags
    • Manage user accounts - block users, change permissions
  3. Administrators
    • Full access to all user accounts - can appoint moderators

Back-end Design

N2 is developed using the Django framework. This is a highly popular Python web development framework, used in well-known online services such as Instagram and Pinterest.

The Django framework follows the Model-View-Controller (MVC) design pattern. The front-end implementation is the View component. We write Python scripts containing the application logic, which is the Controller component. Finally, data is stored in a persistent database, the Model component.

We chose the MySQL database is selected for its high compatibility and ease of maintenance, and the InnoDB storage engine for better consistency.

The database for N2 is carefully designed to meet data storage needs efficiently. It must cover main objects such as users and notices, and also simpler objects including likes and bookmarks.

Security Features

Permission Control
The three levels of user permissions prevent unauthorized changes and broadcast of notices.

University Member Verification
Upon registration, N2 will send an email verification to authenticate membership of the educational institution. Emails can only be sent to addresses within a specific domain which belongs to the university.

Data Protection
Django's built in object-relational mapping system means that all SQL queries are generated by Django. They are automatically escaped, making it injection-proof.

Web Security
N2 utilizes Python libraries and Django's security features to eliminate both XSS and CSRF attacks.

The Result

N2 is a user-customizable web-based announcement board for university students.

It provides a unified notices management experience for all members of the university.

All users need to login, or register if they haven't done so.

Upon registration they will be required to supply a university email address, which will be used to verify membership.

In the homepage, students can see all posts. They can also search based on keywords and tags.

They are able to see any individual post in detail. They can 'like' them, and bookmark them for future reference - these functionalities use Ajax, so only the buttons are updated instead of the entire page being refreshed. The total number of likes are displayed alongside the total number of views.

Anyone can create a post, complete with a display image and video embedding from YouTube or Vimeo. If they have normal permissions, their post will be subject to moderation before it can be visible to others.

Users can configure their preferences for tags of posts that they wish to see by default.

The N2 Administrator interface allows moderators and administrators to manage posts, tags and user permissions.

Feature Extensions For The Future

N2 can be extended to include the following features in the future:

Integration with the university account database. This should enable single sign-in, and eliminate the need to register for accounts.

Security can be improved by using HTTPS, and CAPTCHA can be used to prevent spam.

Students can book into events, and then be reminded of them via e-mail as the events approach. Event organizers can then have access to a guest list, and students can have access to a simple list of all events they have booked for.

The keyword search feature of N2 has a simple implementation at the moment, and the algorithms can be much improved for better performance and accuracy.

Students can mark notices as duplicates of each other, and moderators can monitor this to consolidate redundant posts.

The user interface can be enhanced to support mobile phones and other portable devices.
